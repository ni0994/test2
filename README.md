# 情報セキュリティ 学習アプリ（学習用リポジトリ）

> GitHub 上での検索・ファイル構成整理・PR対応を想定した「学習用アプリ」リポジトリです。  
> 本番用のリリース・コミットは、別リポジトリまたはローカル環境で行います（GitHub でのコミットは最小限に抑えます）。

## 概要

- 情報セキュリティ系試験（例：情報セキュリティマネジメント試験）向けの問題演習アプリ
- HTML・CSS・Vanilla JS で構成し、モジュール分割された構成です
- 一問ごとの解答・回答履歴・ブックマーク・フィルタ・検索・ダッシュボードを備えています
- `type="module"` で ES モジュール構成を採用、`importmap` で `fuse.js` を ESM で利用

### 特徴

- 問題文・選択肢・解説・タグに関する検索（Fuzzy Search 使用）
- 分野／難易度／出題対象（未回答・間違った問題・ブックマーク）によるフィルタ
- 回答履歴の保存（`localStorage`、`debounce` による書き込み）
- ブックマーク機能・選択肢シャッフル
- ランダム10問モード（Fisher–Yates シャッフル）
- 回答率・進捗ダッシュボード（分野別・難易度別）
- 隠し管理パネル（ヘッダー5回クリックで表示）

## ディレクトリ構成

```text
/
├── index.html             ← メイン HTML、入口（importmap 定義）
├── style.css              ← プロジェクトのスタイル
├── manifest.json          ← PWA 用マニフェスト（現状未完成）
├── data/
│   ├── meta.json          ← メタデータ（分野構成等问题集のメタ情報）
│   ├── questions_*.json   ← 問題データ（複数ファイルをマージ）
├── modules/
│   ├── state.js           ← グローバル状態の一元管理（`export let` + setter）
│   ├── storage.js         ← `localStorage` 永続化（`answered` / `bookmarks` + `debounce`）
│   ├── loader.js          ← `fetch` による問題・メタデータ読み込み
│   ├── shuffle.js         ← Fisher–Yates シャッフル関数群
│   ├── search.js          ← Fuse.js 初期化・フィルタ統合（`applyFilters` 等）
│   ├── stats.js           ← 回答統計・ダッシュボード更新
│   ├── render.js          ← `buildCard` / `renderQuestions` で、カードと一覧描画
│   └── ui.js              ← ほとんどすべての `setup*` 関数（UIイベント登録・DOM生成ユーティリティ）
└── app.js                 ← エントリーポイント（`init()` と回答処理・イベントコールバック結びつけ）
```

## 動作環境・前提条件

- ブラウザ：  
  - Chrome 89+ / Safari 16.4+ 等、`type="module"` と `importmap` に対応したブラウザ  
    （IE 非対応）
- サーバ側：  
  - ローカル：  
    - 例：`python -m http.server 8000` / VS Code の Live Server / その他のローカルサーバ  
  - ファイル直接開き（`file://`）は、モジュール読み込みに失敗する可能性があります

## 主な機能と使い方

### 問題表示・検索・フィルタ

- 検索欄にキーワード（問題文・選択肢・解説・タグ）を入力して、問題を絞り込み
- 下部のフィルタメニューで、  
  - 分野・難易度  
  - 出題対象（全て・未回答・間違った問題・一度も正解してない問題・ブックマーク済み）  
  を選択可能
- タグをクリックすると、そのタグに絞り込み（`active-tag` として表示）

### ブックマーク・ランダム10問

- 各問題カード左のブックマークアイコン（📌 / 🔖）をクリックして、問題をブックマーク  
- フィルタの「出題対象」を `ブックマーク済み` にすると、ブックマークした問題だけを表示
- 「ランダム10問」ボタンを押すと、現在のフィルタ条件に当てはまる問題から Fisher–Yates シャッフルで10問を抽出して表示

### 回答方法

- マウスで選択肢ボタン（A/B/C/D）をクリックして回答
- キーボード shortcuts：  
  - A / B / C / D キーで、表示中の一番近い回答未済カードの選択肢をクリック（`setupKeyboardAnswerEvents`）

### ダッシュボード

- 上部の「📊 進捗・正答率」に  
  - 全体の解答数・正解数・正答率・進捗バー  
  - 分野別・難易度別の正答率・進捗バー  
  が表示されます
- 回答を進めるほど、ダッシュボードがリアルタイムに更新されます（`debounce` あり）

### 隠し管理パネル（デバッグ用）

- ヘッダー部分を連続5回クリックすると、**隠し管理パネル**が表示されます
  - キャッシュ削除（Service Worker 系は未実装・現在は `localStorage` 関連の理解用）
  - 回答履歴を全消去
  - 回答履歴のみ消去（ブックマークは保持）
  - パネルを閉じるボタン
- 本番運用ではこの機能を削除または条件付きで非表示にする前提です

## モジュール構成の意図（学習用ポイント）

- **`state.js`**  
  - 全モジュールが共有するグローバル状態の唯一真のソース  
  - 外部から `import` するだけの単純化を意図（`setter` 経由で変更）

- **`storage.js`**  
  - `localStorage` の読み書きと `debounce` を一元化  
  - `LS_ANSWERED_V2` 系のキー名をまとめる役割

- **`loader.js`**  
  - 問題データ・メタデータの読み込みロジック  
  - JSON が不正 / 存在しない場合の `throw` で、`init()` で `catch` して UI に出す構造

- **`shuffle.js`**  
  - Fisher–Yates による選択肢シャッフル  
  - ランダム10問の `shuffle` が一元で同じ関数を使い、統計偏りを抑制

- **`search.js`**  
  - `Fuse.js` の初期化と `applyFilters` で「検索＋分野／難易度／状態フィルタ＋タグフィルタ」を統合  
  - ロジックを一箇所で扱い、テスト容易性を向上

- **`render.js`**  
  - `buildCard` で UI 构成を一元化  
  - `escapeHtml` を含め、DOM 攻撃対策のためのエスケープロジックもそこに集約

- **`stats.js`**  
  - `computeStats` / `updateStatsDashboard` で統計・UI 更新を分離  
  - モジュールとして切り出すことで、他リポジトリに移植しやすい形に

- **`ui.js`**  
  - ほとんどすべての `setup*` 関数を集中  
  - `callbacks` を渡す形で、`app.js` から注入する構造（依存の可視化・テスト容易）

- **`app.js`**  
  - エントリーポイントとして初期化とコールバック結びつけのみ  
  - ビジネスロジックは各モジュールに任せ、`app.js` は glue としての役割にする

## 今後の学習用拡張のアイデア（やる順の例）

1. **PWA の実装**  
   - `service-worker.js` を追加し、`manifest.json` と連携してオフラインキャッシュを実装

2. **仮想スクロール**  
   - 画面内にたくさんの問題が表示される時に、`IntersectionObserver` を使った仮想スクロール化

3. **モード追加（1問1答モード）**  
   - 「Next 問題」形式で、問題を1問ずつ進められるモード

4. **学習履歴の可視化（チャート）**  
   - Chart.js / D3.js で、`history` 配列から日別・週別正答率グラフを追加

5. **`IndexedDB` への移行検討**  
   - 問題数・ユーザー数が増えることを想定した Persistence 層の検討

6. **テストを入れる**  
   - Jest / Vitest で、`shuffle.js` / `stats.js` / `search.js` などの関数をユニットテスト
